# ТЗ: перестройка MOTORA MVP → MOTORHOF CRM

**Для исполнителя (Claude Opus 4.8 / любая модель).** Это управляющий документ. Полные требования заказчика — в `docs/requirements-motorhof.md` (далее «Требования») — читать раздел фазы ПЕРЕД её началом. Конвенции кода — в `CLAUDE.md`, они обязательны. История проекта — в `DEVPLAN.md` (не редактировать, он закрыт; прогресс отмечать здесь).

## Железные правила (нарушение любого = фаза не принята)

1. **Одна фаза за раз.** Не начинать следующую без явной команды пользователя. Фаза заканчивается: рабочим состоянием, прогнанными воротами качества, деплоем на прод, отчётом пользователю.
2. **Ворота качества каждой фазы:** `npx tsc --noEmit && npm run lint && npm run build && npx vitest run` (с фазы 1) — всё чисто. Плюс ручная проверка затронутых сценариев в браузере (локально И на проде после деплоя).
3. **Не заявлять готовым то, что существует только визуально** (Требования §2.14). Кнопка без работающего сервер-экшена — не сделана.
4. **Финансовые вычисления — только в `src/lib/finance.ts`** (сервис с юнит-тестами). В компонентах НИКАКОЙ денежной арифметики, только вызовы сервиса и форматирование.
5. **Деньги — Prisma `Decimal` / Postgres NUMERIC(12,2)**, округление до цента по единому правилу (half-up). JS `number` для денег запрещён в постоянном хранении и расчётах (Требования §2.7).
6. **Миграции — только `prisma migrate`** (переход с `db push` — фаза 1). Перед каждой миграцией схемы: бэкап (`pg_dump` через Neon или экспорт-скрипт). Миграции повторяемы, без orphan records.
7. **Legacy не удалять физически.** Deal, объём двигателя и прочее наследие — soft-выключение и/или legacy-таблицы до ручной проверки (Требования §4, §22).
8. **Права — server-side.** Скрытая кнопка без проверки в Server Action — не защита (Требования §20).
9. **Git:** ветка на фазу → ff-merge в `main` → push (авто-деплой Vercel). Сообщения на русском: `фаза 2: аккаунты и роли`.
10. Каждый новый CSS — внутри `@layer` (см. CLAUDE.md, критично для Tailwind v4).

## Решения, принятые до старта (не пересматривать без пользователя)

| Вопрос | Решение |
|---|---|
| Валюта | EUR, формат `Intl.NumberFormat('de-AT')` → `€ 12.500,00`. Существующие числа НЕ конвертировать по курсу — перечитать как EUR |
| Даты / TZ | `DD.MM.YYYY`, Europe/Vienna |
| Язык UI | Русский; австрийские термины по-немецки: Pickerl, Differenzbesteuerung, Gewährleistung, Kaufvertrag… |
| «Дашборд» | Оставить как есть (Требования используют это слово) |
| Палитра | Убрать доминирующий янтарь. База: Graphit `#3B3F42`, Warm White `#F6F3F2`, White `#FFFFFF`, тёмный фон остаётся. Зелёный/красный/жёлтый/синий — только мелкие функциональные акценты |
| Статусы в БД | Остаются строковыми enum-ами (native enum Postgres не вводить — лишний риск миграций). Валидация через словари `format.ts` + TS-типы |
| Подпись под лого | Оставить «CRM автосалона» (решение 20.07) |
| Хранилище файлов | **Cloudflare R2** через S3-совместимую абстракцию (`src/lib/storage.ts`). ЖЁСТКО: никакого прямого API R2 в компонентах — только абстракция. Причина: в будущем переезд на Hetzner+MinIO (тот же S3-протокол) сменой env, без переписывания |
| Логотип | SVG получен: `MOTORHOF_Symbol_Primary_Graphite_3B3F42.svg` (монограмма MH, штрих #3B3F42). Форму не менять; цвет штриха — через CSS/currentColor (на тёмном фоне Warm White, на светлом графит) — это адаптация цвета, не деформация |
| Роли | ФИНАЛЬНАЯ модель (20.07, см. `docs/roles-motorhof.md`): **ADMIN / PARTNER / SALES / TECHNICAL / READ_ONLY**, пользователь может иметь несколько ролей (права = union). Проверки через `can(user, capability)` по капабилити-матрице, НЕ `if role===`. Ключевое: видимость финансов скрывается **на сервере** (`redact`), не в UI |
| Данные на проде | Тестовые (подтверждено 20.07) — миграции фазы 1 можно гонять свободно, полный re-seed допустим |
| Hetzner | Есть арендованный сервер. Сейчас НЕ используем (Vercel+Neon+R2 = $0 и ноль обслуживания). План: при передаче в боевую эксплуатацию — переезд всего стека на Hetzner (Docker: app + Postgres + MinIO), снимает вопрос некоммерческого тарифа Vercel и сна Neon |
