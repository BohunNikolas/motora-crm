## Модель данных (prisma/schema.prisma — создана)

- **Car** — авто: make, model, year, mileage, vin?, color?, transmission?, fuel?, engineVol?, purchasePrice (закупка), listPrice (цена продажи), status (`PREP|AVAILABLE|RESERVED|SOLD`), notes?. Связи: expenses[], deals[], tasks[]
- **Expense** — расход на подготовку авто: title, amount, date, carId (cascade delete)
- **Client** — клиент: name, phone, email?, type (`BUYER|SELLER|BOTH`), source? (авито/авто.ру/рекомендация/сайт/проходящий), notes?. Связи: deals[], tasks[]
- **Deal** — сделка: type (`SALE|PURCHASE|TRADE_IN`), stage (`NEW|CONTACT|TEST_DRIVE|NEGOTIATION|CONTRACT|DONE|LOST`), amount?, notes?, closedAt?, carId?, clientId
- **Task** — задача: title, dueDate?, done, carId?, clientId?, dealId?

Бизнес-правила:
1. Себестоимость авто = purchasePrice + Σ expenses.
2. Ожидаемая маржа = listPrice − себестоимость; фактическая = deal.amount − себестоимость.
3. Перевод сделки в DONE (кроме type=PURCHASE) → авто автоматически `SOLD`, ставится closedAt. Возврат из DONE → авто `AVAILABLE`.
4. Выручка месяца = Σ amount сделок DONE с closedAt в текущем месяце (без PURCHASE).

## Известные грабли

- `command not found: npx` → забыл `export PATH="$HOME/.local/node/bin:$PATH"`
- Prisma 7 ставится по умолчанию — всегда `npm i prisma@6 @prisma/client@6`
- В Next 16 `params` — Promise: `const { id } = await params`
- Tailwind v4: конфиг в CSS (`@theme inline`), файла tailwind.config нет — токены добавлять в globals.css
- SQLite не поддерживает enum — статусы это String, валидация на уровне словарей format.ts
- Названия папки проекта содержит кириллицу (`клод`) — в bash-командах всегда брать путь в кавычки
- **CSS без `@layer` перебивает утилиты Tailwind v4** — самый коварный баг проекта, см. CLAUDE.md
- `.claude/launch.json` для dev-сервера лежит в корне рабочей папки (`/Users/nikolas77/Desktop/клод/`), а не в `autocrm/`. Запуск обёрнут в `/bin/sh -c` с экспортом PATH, иначе `npm` не находит node
- Поиск по кириллице: SQLite не умеет регистронезависимый LIKE для не-ASCII, поэтому текстовый поиск в `/cars` сделан фильтром в JS. После переезда на Postgres (этап 8) можно вернуть `contains` + `mode: "insensitive"`
- Панель браузера в этой среде не доставляет клики/клавиатуру в приложение (скриншоты, `form_input`, JS и навигация работают). Для проверки Server Actions использовать `form.requestSubmit()` через javascript_tool и сверять вызовы в `preview_logs`
